# 🏬 뜨개 공방/스토어 전용 CRM (KnitStore Manager) 기획안

## 1. 개요 (Overview)
- **가칭:** 니트스토어 매니저 (KnitStore Manager)
- **비전:** 일반 POS/CRM으로는 해결할 수 없는 '로트 번호(Dye Lot)' 기반의 까다로운 실 재고 관리와 수강생의 뜨개질 진도율 관리를 단 하나의 플랫폼에서 통합으로 제공하는 맞춤형 B2B SaaS입니다.
- **타겟 고객:**
  1. 오프라인 뜨개 공방을 운영하는 강사
  2. 로컬에서 실과 부자재를 판매하는 뜨개 스토어 오너

---

## 2. 해결하고자 하는 문제 (Problem)
### 기존 매장 관리(POS/CRM) 시스템의 한계
- **로트(Dye Lot) 관리 불가:** 뜨개실은 **'동일한 브랜드, 동일한 색상 번호'**라도 염색 차수(로트)가 다르면 미세한 톤 차이가 납니다. 옷을 뜨는 중간에 실이 모자라 다른 로트의 실을 이어 붙이면 확연히 티가 나서 완성품을 망치게 됩니다. 일반적인 POS는 바코드 단위(품번)까지만 관리하므로 로트 단위의 정밀한 재고 관리가 불가능합니다.
- **수강생 현황 파악의 어려움:** 수강생들은 보통 여러 개의 도안을 동시에 뜨고(문어발), "선생님, 저 이거 어디까지 떴죠?" 하고 묻는 경우가 허다합니다. 강사들이 수강생 개개인의 도안, 진도율, 수강권 차감 횟수를 엑셀이나 수첩에 일일이 기록해야만 했습니다.

---

## 3. 핵심 기능 (Core Features)

### ① 로트(Lot) 번호 기반 정밀 재고 관리
- **다중 로트 분리 관리:** 특정 품번(예: 알리제 털실 레드 #101) 입고 시, 로트 번호(Lot: 1234, Lot: 5678)별로 재고 수량을 분리해서 관리할 수 있습니다.
- **스마트 판매 알림:** 수강생이나 손님이 스웨터 하나를 뜰 분량(10볼)을 살 때, 동일한 로트 번호가 10볼이 안 되면 "로트 번호가 달라 섞여 판매될 수 있습니다"라고 경고창을 띄워 오안내를 방지합니다.

### ② 수강생(Class) 및 진도율 통합 관리
- **회원별 프로젝트(도안) 대시보드:** A 수강생이 어떤 도안(예: 가디건, 비니)을 배우고 있는지, 현재 진도(예: "소매 분리 직전")와 사용한 실의 로트 번호를 강사가 CRM에서 클릭 한 번으로 확인하고 기록합니다.
- **수강권 자동 차감:** 클래스 예약 및 출석 시 보유 수강권이 자동 차감되며, 남은 횟수를 수강생에게 카카오톡(알림톡) 등으로 자동 전송합니다.

### ③ 판매/수강 일원화 POS 화면 (태블릿 최적화)
- 실 판매와 수강료 결제를 동시에 처리할 수 있는 직관적인 웹 기반 태블릿 POS UI를 제공합니다. 브라우저 카메라 기능(WebRTC)을 활용해 별도 바코드 스캐너 기기 없이도 태블릿/스마트폰 카메라로 실의 바코드를 스캔할 수 있습니다.

## 4. 수익 모델 (Monetization - SaaS)

- **강사 및 1인 공방 플랜 (Basic):** 월 $19 (또는 월 2만원대)
  - 특징: 수강생 30명 이하, 한정된 품목의 재고 관리, 모바일/태블릿 접속
- **로컬 스토어 및 대형 센터 플랜 (Pro):** 월 $49 (또는 월 5만원대)
  - 특징: 수강생 Му제한, 무제한 재고 등록, 바코드 연속 스캔, 멀티 디바이스(직원별 계정 생성), 알림톡 API 연동 무료 제공

## 5. MVP (최소 기능 제품) 개발 로드맵

### Phase 1: 로트 기반 인벤토리 시스템 (1~2주)
- [ ] 품목 및 로트 번호 단위로 관리되는 데이터베이스 구조(Schema) 설계
- [ ] 상품 등록/입고 및 로트별 재고 목록 확인이 가능한 어드민 웹 페이지 구축

### Phase 2: 수강생 관리 및 대시보드 (2~4주)
- [ ] 수강생 개인 프로필 페이지 구축 (수강 정보, 구매한 실 내역 기록)
- [ ] 강사가 직접 진도율과 코멘트를 메모 형식으로 남길 수 있는 기능
- [ ] 출석 버튼 클릭 시 수강 횟수 자동차감 로직 구현

### Phase 3: 연동 및 테스트 (4~6주)
- [ ] 반응형 UI 적용으로 아이패드 및 모바일 웹에서도 편하게 사용 가능하도록 최적화
- [ ] 결제액/수강료 월별 매출 통합 리포트를 보여주는 대시보드 기능 (차트)
- [ ] 실 사용 공방 대상 1개월 무료 베타 테스트 진행

## 6. 권장 기술 스택 (Tech Stack)
- **Frontend:** Next.js (TypeScript) + Shadcn + Tailwind CSS - 모바일/태블릿에서도 가벼운 반응형 UI를 빠르게 구축하는 데 유리합니다.
- **Backend/DB:** Supabase - 복잡한 RDBMS 구조(회원 ↔ 수강생 ↔ 재고 ↔ 결제 연결)가 필요하므로 PostgreSQL 기반의 Supabase가 백엔드 서버 없이도 강력한 성능을 냅니다.
- **UI Library:** shadcn/ui 기반 컴포넌트를 사용하여 POS 기기와 유사한 직관적인 버튼과 폼을 빠르게 설계합니다.
- **Scanner:** HTML5 바코드 스캐너 라이브러리(ZXing) 활용 - 카메라 권한으로 바코드 웹 리딩.
